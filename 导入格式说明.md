# MindFlow 导入格式说明

> 本文档定义 MindFlow 思维导图工具的**标准导入格式**。
> 无论你用什么 AI（豆包 / ChatGPT / Claude / Gemini / 文心一言）或什么编辑器（WPS / Typora / Notion 导出），只要按此格式生成内容，导入后导图结构就能完整保留。

---

## 标准格式：Markdown 大纲

MindFlow 使用 **Markdown 大纲**作为标准格式。这是最通用、所有 AI 和编辑器都能输出的格式。

### 基本规则

```
# 根节点（主题，只能有一个）

## 一级分支（核心板块，3-7 个）

- 二级要点（每个分支下的细节）
- 二级要点
  - 三级细节（可选，缩进 2 空格）
```

### 完整示例

```markdown
# 三农核心知识

## 一、乡村振兴战略

- 总目标：农业农村现代化
- 总方针：坚持农业农村优先发展
- 总要求：产业兴旺、生态宜居、乡风文明、治理有效、生活富裕
- 五个振兴：产业、人才、文化、生态、组织

## 二、农村土地制度

- 三权分置：所有权、承包权、经营权
- 承包期限
  - 耕地：30 年
  - 草地：30-50 年
  - 林地：30-70 年
- 耕地保护红线：永久基本农田不得改变用途

## 三、农业产业发展

- 三区三园一体
- 产业融合：农文旅三位一体
- 农民专业合作社：一人一票制
```

---

## 层级对应关系

| Markdown | 导图层级 | 用途 | 数量建议 |
|---|---|---|---|
| `#` | 根节点 | 主题（只 1 个） | 1 个 |
| `##` | 一级分支 | 核心板块 | 3-7 个 |
| `-` | 二级要点 | 具体内容 | 每分支 3-8 条 |
| `  -`（缩进） | 三级细节 | 补充说明 | 可选 |

---

## 支持的输入方式

### 方式 1：导入 `.md` 文件（推荐）

用 AI 或编辑器生成 `.md` 文件 → 点 MindFlow 顶栏"导入"按钮 → 选择文件。

**自动检测**：
- 内容含 `#` 标题 + `##`/`-` 列表 → **直接生成导图**（不走 AI，结构原样保留）
- 纯文本/文章 → 走 AI 整理

### 方式 2：导入 `.txt` 文件

`.txt` 文件会被当作自然语言，交给 AI 整理成导图。

### 方式 3：IDE 协同推送

在 IDE 里让 AI 生成标准格式 markdown，通过 `push-markmap` 接口直提。

## 各 AI 的提示词模板

### 豆包 / ChatGPT / Claude / Gemini 通用提示词

```
请把以下内容整理成 Markdown 大纲格式的思维导图。

格式要求：
1. 第一行用 # 作为根节点（主题）
2. 用 ## 作为一级分支（3-7 个核心板块）
3. 用 - 作为二级要点（每个板块 3-8 条）
4. 缩进的 - 作为三级细节（可选）
5. 不要加粗、链接、图片、代码块，只用纯文本
6. 直接输出 Markdown，不要加解释

内容：
[在这里粘贴你的内容]
```

### WPS / Typora / Notion 用户

直接用编辑器的导出为 Markdown 功能，确保：
- 有一个 `#` 根标题
- 板块用 `##`
- 要点用 `-`

---

## 格式对照表（常见 AI 输出 → MindFlow）

| AI/工具的输出格式 | 能否直接导入 | 说明 |
|---|---|---|
| `#`/`##`/`-` Markdown 大纲 | ✅ **直接导入** | 标准格式，原样保留 |
| `*` 列表（非 `-`） | ✅ 直接导入 | `*` 也被识别为列表 |
| `1.` `2.` 数字列表 | ✅ 直接导入 | 数字编号被识别为列表 |
| 树形字符 `├─` `│` `└─` | ⚠️ 走 AI 清洗 | 自动清洗树形字符后交 AI |
| 纯文本段落（无标记） | ⚠️ 走 AI 整理 | AI 重新理解并生成结构 |
| Word/PDF/图片 | ❌ 不支持 | 先转成 `.md` 或 `.txt` |

## 常见问题

### Q: 导入后结构不对/丢了内容？

检查你的 `.md` 文件是否符合标准格式：
- [ ] 有且只有一个 `#`（根节点）
- [ ] 板块用 `##`（不是加粗或其他）
- [ ] 要点用 `-`（不是 `•` 或其他符号）
- [ ] 没有嵌套代码块（```）

### Q: 我的 AI 生成的格式和标准不一样？

用上面的**通用提示词模板**，明确告诉 AI 输出 `#`/`##`/`-` 格式。

### Q: 能不能导入已经是思维导图的文件（如 .xmind）？

暂不支持。先用 XMind 导出为 Markdown，再导入。

## 技术细节（给开发者）

检测逻辑见 `shared/markdownImport.ts` 的 `analyzeMarkdownOutline`：
- `#` 标题 ≥2 或 `#` + 列表 → 直提（push-markmap）
- 纯列表 ≥4 项 → 直提 + 自动补根标题
- 其他 → 走 AI/本地引擎

标准格式定义在此文档，所有 AI 提示词模板也以此为准。
